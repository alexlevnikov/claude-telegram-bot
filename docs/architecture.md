## Описание каждого компонента

### Основной бот (screen `claude`)

Это главный рабочий процесс. Claude Code запускается в сессии `screen` с именем `claude`, в рабочей директории `~/Documents/Projects`.

Ключевые параметры запуска:
- `--remote-control "Build Server"` — включает remote-control интерфейс, через который хуки могут инжектировать команды
- `--channels "plugin:telegram@claude-plugins-official"` — подключает официальный Telegram-плагин, который читает сообщения и даёт Claude инструменты `reply`, `react`, `download_attachment`
- `caffeinate -dimsu` — предотвращает засыпание Mac пока Claude работает

Инструкции для поведения бота хранятся в `~/Documents/Projects/CLAUDE.md`: форматирование ответов в Telegram, обработка голосовых сообщений, кнопки меню, обработка slash-команд.

### Бот Вовка (screen `claude-mama`)

Изолированный Claude Code с отдельным Telegram-токеном и отдельной рабочей директорией `~/Documents/MamaBot`. Работает та же система хуков, но с другим `TELEGRAM_STATE_DIR` — поэтому токены и allowlist берутся из `~/.claude/channels/telegram-mama/`.

Бот имеет тёплую персону (Вовка из мультфильма), адаптирован для нетехнической пользовательницы Светланы. Знает двух пользователей по `chat_id`: Светлана (основная) и Алекс (администратор с расширенными правами).

### Контрол-бот (`commands.ts`)

Лёгкий процесс на Bun (~180 строк), независимый от Claude Code. Работает с третьим Telegram-токеном.

**Зона ответственности контрол-бота:**
- Получать и обрабатывать callback-кнопки от Stop-хука (`bs:continue`, `bs:retry`, `bs:stop`)
- Получать и обрабатывать кнопки stuck-алертов (`bs:restart`)
- Отвечать на `/restart`, `/restart_mama`, `/status`
- Получать уведомления от хуков и `new-session.sh` (🔴/🟢 онлайн/офлайн, ✅/❌ готово)

**Почему отдельный бот:** inline-кнопки привязаны к боту, который их отправил. Если бы уведомления отправлял основной бот, он же должен был бы и обрабатывать нажатия кнопок — но у плагина уже есть свой long-polling loop, и два поллера на одном токене конфликтуют. Контрол-бот решает эту проблему.

**Важно:** контрол-бот работает даже когда у Claude Code закончились кредиты или сессия мертва. Он не требует Claude — только Bun и токен.

### Хуки Claude Code

Хуки — скрипты, которые Claude Code запускает при определённых событиях. Настроены в `~/.claude/settings.json`.

#### `UserPromptSubmit` → `prompt-submitted.ts`

Срабатывает когда пользователь отправляет промпт. Делает две вещи:
1. Создаёт файл `~/.claude/buildserver/state/prompt-active.json` с временной меткой и путём к транскрипту — stuck-детектор использует этот файл
2. Отправляет в Telegram сообщение «🤔 Думаю…» через основной бот — чтобы пользователь видел, что запрос принят

#### `PostToolUse` → `progress.ts`

Срабатывает после каждого использования инструмента. Редактирует сообщение «🤔 Думаю…» на актуальный прогресс:
- `📖 читает файл /путь/к/файлу`
- `⚙️ выполняет команду npm test`
- `🌐 загружает страницу`
- `✏️ редактирует файл`

Ключ состояния: `chatId_threadId` (для forum-супергрупп) или `chatId` (для личных чатов). Хранится в `/tmp/claude-tg-progress-<ключ>.json`.

#### `Stop` → `on-stop.ts` → `notify-stop.ts` + `compact-check.ts`

Срабатывает когда Claude завершил ход. `on-stop.ts` — оркестратор: читает stdin один раз и передаёт payload обоим скриптам (иначе второй скрипт получил бы пустой stdin из-за особенности pipe в bash).

**`notify-stop.ts`:**
- Удаляет сообщение с прогрессом из Telegram
- Удаляет `prompt-active.json` (снимает флаг «Claude занят»)
- Читает последний ход ассистента из транскрипта
- Если Claude ответил через Telegram (`reply` tool) — не дублирует
- Если Claude работал без Telegram-ответа — отправляет ✅/❌ уведомление через контрол-бот с кнопками ▶ Продолжить / 🔁 Повторить / 🛑 Стоп

**`compact-check.ts`:**
- Читает использование токенов из транскрипта
- Вычисляет заполненность контекста: `(input_tokens + cache_creation_input_tokens) / context_window * 100%`
- При ≥ 35% (COMPACT_THRESHOLD) — инжектирует `/compact` и уведомляет
- При ≥ 80% (CLEAR_THRESHOLD) — инжектирует `/clear` и уведомляет

### `launchd-boot.sh` и watchdog

Запускается launchd каждые 60 секунд. Выполняет три проверки:

1. **Stuck-детектор:** если `prompt-active.json` старше `STUCK_MINUTES` (15) минут — отправляет алерт через контрол-бот и создаёт флаг `.alerted` (чтобы не спамить). Флаг убирается когда Claude наконец ответит (в `notify-stop.ts`).

2. **Контрол-бот:** проверяет что `commands.ts` жив по PID-файлу. Если мёртв — перезапускает.

3. **Авто-recovery:** если screen-сессия `claude` не найдена — поднимает заново и отправляет «🔄 Билд-сервер перезапустился» через контрол-бот.

### `new-session.sh`

Скрипт для создания новой сессии. Вызывается самим Claude (из CLAUDE.md) при `/newsession`.

Флоу намеренно асинхронный: Claude запускает скрипт и он сразу возвращает `"scheduled:main:delay=4s"`, не блокируя текущий ход. Фоновый процесс через 4 секунды:
1. Шлёт «🔴 Билд-сервер: перезапуск — ухожу в оффлайн»
2. Убивает screen-сессию и запускает новую
3. Ждёт до 60 секунд пока сессия поднимется
4. Перерегистрирует команды Telegram (плагин сбрасывает их при запуске)
5. Шлёт «🟢 Билд-сервер: снова онлайн»

## Поток данных: от сообщения до ответа

```
1. Пользователь пишет в Telegram
          │
          ▼
2. Telegram Bot API доставляет update плагину
   (плагин делает long polling внутри Claude Code)
          │
          ▼
3. Плагин оборачивает сообщение в тег <channel source="telegram" chat_id="..." ...>
   и передаёт в контекст Claude
          │
          ▼
4. Claude Code триггерит UserPromptSubmit хук:
   → prompt-submitted.ts создаёт prompt-active.json
   → отправляется «🤔 Думаю…» в Telegram
          │
          ▼
5. Claude обрабатывает задачу, использует инструменты
   Каждый инструмент: → PostToolUse хук → progress.ts
   → «🤔 Думаю…» обновляется на «📖 читает файл…»
          │
          ▼
6. Claude использует инструмент reply() для ответа
   (или отвечает только через Stop-хук если задача не требовала reply)
          │
          ▼
7. Claude Code триггерит Stop хук:
   → on-stop.ts читает stdin
   → notify-stop.ts:
       • удаляет прогресс-сообщение
       • удаляет prompt-active.json
       • если Claude не отвечал через reply — шлёт ✅/❌ через контрол-бот
   → compact-check.ts:
       • считает % контекста
       • при превышении — инжектирует /compact или /clear
          │
          ▼
8. Ответ в Telegram, кнопки ▶/🔁/🛑
```

## Почему два бота, а не один

| Причина | Объяснение |
|---|---|
| **Изоляция контекста** | У каждого бота своя история разговоров, свои CLAUDE.md, свои проекты. Задачи разработки не смешиваются с личными вопросами Светланы. |
| **Разные personas** | Основной бот — технический, делает git, запускает тесты. Вовка — тёплый, бытовой, скрывает технику. Смешивать нельзя. |
| **Разные права и настройки** | Основной бот: модель opus, широкие разрешения на Bash. Вовка: sonnet, ограниченные разрешения, своё `settings.json`. |
| **Разные rабочие директории** | `~/Documents/Projects` vs `~/Documents/MamaBot` — Claude Code привязан к cwd, смешивать опасно. |
| **Независимость** | Если основной бот завис или тратит кредиты на тяжёлую задачу, Вовка продолжает работать. |

## Жизненный цикл контекста

Контекст Claude Code — это окно токенов, которое заполняется по ходу разговора.

```
0%                   35%                  80%              100%
│────────────────────│────────────────────│────────────────│
│  Нормальная работа │  /compact (сжатие) │  /clear (сброс)│
```

**Нормальная работа (0–35%):** контекст накапливается. Каждое сообщение, каждый файл, каждый ответ добавляют токены.

**Compact-зона (35–80%):** при остановке Claude хук `compact-check.ts` инжектирует `/compact`. Claude Code делает резюме разговора и сбрасывает детальную историю, сохраняя суть. Контекст уменьшается примерно до 10–15%.

**Критическая зона (≥80%):** это происходит когда задача долгая — Stop-хук не срабатывает пока Claude не закончил, а за это время контекст может вырасти с 35% до 90%+. При ≥80% хук инжектирует `/clear` — полный сброс без резюме.

**Расчёт процента:** используется `input_tokens + cache_creation_input_tokens`, делённое на `context_window_tokens`. `cache_read_input_tokens` намеренно исключён — он может быть больше окна из-за многоуровневого кэша Claude, что давало ложные 150–200%.

Подробнее: [context-management.md](./context-management.md).

## Детальная схема хуков

```
Событие: UserPromptSubmit
─────────────────────────
Claude Code получает промпт
       │
       └─► prompt-submitted.ts (bun, timeout 3.5s)
               • Пишет prompt-active.json (для stuck-детектора)
               • Извлекает chat_id из промпта или транскрипта
               • Отправляет «🤔 Думаю…» через основной бот
               • Сохраняет msg_id в /tmp/claude-tg-progress-<chatId>.json

Событие: PostToolUse (на каждый инструмент)
────────────────────────────────────────────
Claude использовал инструмент
       │
       └─► progress.ts (bun)
               • Читает транскрипт → находит последний chat_id
               • Определяет label по названию инструмента (Read→📖, Bash→⚙️ и т.д.)
               • editMessageText: обновляет «🤔 Думаю…» → «📖 читает файл/путь»
               • Telegram инструменты (reply, react) → пропускает, чтобы не шумело

Событие: Stop (Claude завершил ход)
────────────────────────────────────
       │
       └─► on-stop.ts (bun)  ← читает stdin один раз
               │
               ├─► notify-stop.ts
               │       • Удаляет все /tmp/claude-tg-progress-*.json
               │       • deleteMessage: убирает прогресс из чата
               │       • Удаляет prompt-active.json + .alerted
               │       • Пишет state/last-stop.txt
               │       • Читает последний ход из транскрипта
               │       • Если reply-инструмент уже был → не дублируем
               │       • Иначе → sendMessage ✅/❌ + кнопки через контрол-бот
               │
               └─► compact-check.ts
                       • Читает usage из последней строки транскрипта
                       • Считает pct = (input + cache_creation) / window
                       • pct ≥ 35% → уведомление + /compact в screen
                       • pct ≥ 80% → уведомление + /clear в screen
```

## Конфигурационный граф

```
~/.claude/settings.json  (глобальные настройки Claude Code)
    ├── model: claude-opus-4-5
    ├── hooks:
    │     UserPromptSubmit → bun ~/.claude/buildserver/prompt-submitted.ts
    │     PostToolUse      → bun ~/.claude/buildserver/progress.ts
    │     Stop             → bun ~/.claude/buildserver/on-stop.ts
    └── permissions: allow/deny списки

~/.claude/channels/telegram/.env
    └── TELEGRAM_BOT_TOKEN  ← используется плагином + tglib.ts

~/.claude/channels/telegram/access.json
    └── allowFrom: ["chat_id"]  ← используется плагином + tglib.ts

~/.claude/buildserver/control.env
    └── TELEGRAM_CONTROL_BOT_TOKEN  ← используется контрол-ботом + хуками

~/Documents/MamaBot/.claude/settings.json
    └── Отдельные настройки Вовки (sonnet, ограниченные права)
```
