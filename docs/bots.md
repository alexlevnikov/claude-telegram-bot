# Использование ботов

---

## Основной бот (Claude Build Server)

### Кому доступен

Только пользователям из allowlist (`~/.claude/channels/telegram/access.json`). Незарегистрированные пользователи игнорируются на уровне плагина.

### Slash-команды

Команды появляются в меню Telegram при нажатии `/`.

#### `/status`

Показывает текущее состояние сервера.

Что отображается:
- Список screen-сессий (`screen -ls`)
- Текущая модель из `settings.json`
- Время последнего ответа (`state/last-stop.txt`)

Пример ответа:
```
📊 Статус

Сессии:
  2345.claude (Detached)
  2346.claude-mama (Detached)

Модель: claude-opus-4-5

Последний ответ: 02.06.2026, 14:23:11
```

#### `/logs`

Последние 20 строк экрана screen-сессии.

Выполняется:
```bash
screen -S claude -X hardcopy /tmp/cl_log.txt
tail -20 /tmp/cl_log.txt
```

Полезно для диагностики когда Claude работает над задачей и нужно посмотреть что происходит на терминале.

#### `/compact`

Сжимает контекст текущей сессии через инжекцию команды:
```bash
screen -S claude -X stuff '/compact\r'
```

Claude Code создаёт резюме текущего разговора и сбрасывает детальную историю. Контекст уменьшается примерно с 35–50% до 10–15%. Содержательная информация сохраняется.

**Когда использовать:** если хотите освободить контекст не теряя нить разговора.

#### `/clear`

Полностью очищает контекст. Claude забывает всё — историю, файлы, переменные разговора.

Перед выполнением Claude спросит подтверждение: «Очистить контекст? Ответь 'да' для подтверждения».

```bash
screen -S claude -X stuff '/clear\r'
```

**Когда использовать:** когда задача полностью завершена и нужно начать новую с чистого листа.

#### `/newsession`

Полный сброс — убивает screen-сессию и запускает новую. Чистый контекст + все ресурсы Claude освобождены.

Флоу:
1. Claude отвечает «🆕 Создаю новую сессию — бот будет недоступен ~15 сек»
2. Запускается `new-session.sh main`
3. Через 4 секунды: 🔴 уведомление об офлайне через контрол-бот
4. Сессия убивается и создаётся новая
5. После поднятия: 🟢 уведомление через контрол-бот

Во время перезапуска (~15 секунд) бот не отвечает.

**Когда использовать:** когда `/compact` и `/clear` уже не помогают, или нужен свежий старт с другим проектом.

#### `/model`

Показывает текущую модель и предлагает изменить через кнопки:

```
⚡ Haiku   ⚖️ Sonnet   🧠 Opus
```

При выборе — обновляет `~/.claude/settings.json` и отвечает:
```
✅ Модель: claude-haiku-4-5. Применится при следующем запуске.
```

Текущая сессия продолжает работать на старой модели. Изменение вступает в силу после `/newsession` или перезапуска.

#### `/projects`

Читает `~/.claude/buildserver/project-map.json` и показывает список проектов с путями. Полезно чтобы быстро переключиться между задачами.

#### `/help`

Список всех команд с коротким описанием.

---

### Reply keyboard (кнопки меню)

При `/menu`, `/start` или слове «меню» — показывается reply keyboard. Кнопки шлют текстовые сообщения, которые Claude обрабатывает:

```
[ 📊 Статус  ] [ 📋 Логи      ]
[ 🗜 Compact  ] [ 🧹 Clear     ]
[ 🆕 Новая   ] [ 🤖 Модель    ]
[ 📁 Проекты ] [ ❓ Помощь    ]
```

Меню «приклеено» к полю ввода (`persistent: true, resize_keyboard: true`).

### Отправка задач

Просто пишите задачу как обычное сообщение:

```
Посмотри ошибки в последнем тесте и исправь
```

```
Сделай git commit с описанием последних изменений
```

```
Покажи структуру директории /src
```

Claude видит сообщение через тег `<channel source="telegram" chat_id="..." user="...">` и отвечает через инструмент `reply`.

**Индикатор прогресса:** пока Claude работает, в чате появляется и обновляется сообщение:
- `🤔 Думаю…` — Claude начал обработку
- `📖 читает файл src/index.ts…` — читает файл
- `⚙️ выполняет команду npm test…` — запускает команду
- `✏️ редактирует файл package.json…` — редактирует файл

По завершении сообщение прогресса удаляется.

---

### Голосовые сообщения

Отправьте голосовое сообщение в Telegram — Claude его расшифрует и ответит.

Флоу обработки:
1. Плагин передаёт `attachment_kind="voice"` и `attachment_file_id`
2. Claude вызывает `download_attachment` → получает путь к `.ogg` файлу
3. Запускает `~/.claude/buildserver/transcribe.sh "<путь>"` → текст
4. Обрабатывает как обычный запрос

**Голосовой ответ** (если настроен TTS): при соблюдении трёх условий:
- Входящее было голосовым
- Ответ короче 500 символов
- Ответ не содержит блоков кода

Claude запускает `tts.sh` и отправляет `.ogg` файл вместе с текстом.

### Уведомления через контрол-бот

После каждого хода Claude (Stop-хук) через контрол-бот приходит уведомление:

**✅ Готово — ProjectName**
```
Реализовал функцию авторизации. Добавил валидацию JWT токена
в middleware, написал 3 теста. Все проходят.
```

**❌ Ошибка — ProjectName**
```
npm test завершился с exit code 1. Тест AuthMiddleware.test.ts
падает на проверке истёкшего токена.
```

Под сообщением кнопки:

| Кнопка | Действие |
|---|---|
| ▶ Продолжить | Отправляет Enter в screen-сессию (Claude продолжает работу) |
| 🔁 Повторить | Инжектирует «Попробуй снова» — Claude перезапускает последнее |
| 🛑 Стоп | Отправляет Escape в screen-сессию — прерывает текущую операцию |

**Если Claude уже ответил через `reply` в Telegram** — уведомление не дублируется. Контрол-бот присылает уведомление только когда Claude работал «молча» (фоновые задачи, длинные операции).

## Бот Вовка

### Для кого

Бот обслуживает двух пользователей с разными ролями:

- **Светлана** (chat_id `1332813880`) — хозяйка бота, нетехнический пользователь. Получает тёплую персону Вовки.
- **Алекс** (chat_id `289326333`) — администратор, сын Светланы. Деловой режим, доступ к техническим командам.

### Режим Светланы

Светлана общается с Вовкой как с личным помощником. Бот:
- Говорит тепло, на «вы», простыми словами
- Не показывает ошибки, пути к файлам, технические детали
- Помнит её имя, предпочтения, важные даты — из памяти Claude
- Отвечает голосом если она написала голосом (при соблюдении условий)
- Даёт шаги по одному, не «вываливает» всё сразу

Примеры запросов Светланы:
```
Как найти рецепт борща?
```
```
Напомни мне завтра позвонить дочке в 3 часа
```
```
Объясни что такое «обновление телефона» — надо ли его делать?
```

### Режим администратора (Алекс)

Когда пишет Алекс — бот переключается в деловой режим.

**Slash-команды (при `/menu` или `/help`):**

```
[ 📊 Статус        ] [ 🆕 Новая сессия ]
[ 🗜 Compact        ] [ 📋 Последний разговор ]
[ ❓ Помощь         ]
```

#### 📊 Статус

```bash
screen -ls | grep claude-mama
cat ~/Documents/MamaBot/.claude/settings.json
```

#### 🆕 Новая сессия

Перезапускает бота Вовку. Флоу аналогичен основному боту: `new-session.sh mama`.

#### 🗜 Compact

```bash
screen -S claude-mama -X stuff '/compact\r'
```

#### 📋 Последний разговор

Вовка кратко рассказывает из памяти о последних разговорах со Светланой:
- что она спрашивала
- что Вовка сделал
- что узнал нового о ней
- есть ли проблемы

#### Прямые вопросы Алексу

Алекс может напрямую спрашивать:
```
Что Светлана спрашивала сегодня?
```
```
Поправь своё поведение — она не любит длинные ответы
```
```
Запомни что у неё день рождения 15 марта
```

### Алерты администратору

Если в разговоре со Светланой происходит что-то важное, Вовка автоматически отправляет Алексу (chat_id `289326333`) короткое уведомление:

Триггеры:
- Светлана расстроена, встревожена, просит срочную помощь
- Тема здоровья, денег, безопасности
- Техническая ошибка (не смог выполнить, не работает голос)

Пример алерта:
```
[Светлана] Расстроена — спрашивала о списании денег с карты.
Нужна ли помощь?
```

Светлане об этих сообщениях Вовка не говорит.

## Контрол-бот

Контрол-бот (`commands.ts`) — это третий, самостоятельный бот. Работает даже когда Claude Code не запущен.

### Команды контрол-бота

#### `/status`

Быстрая проверка живости screen-сессий:

```
🟢 claude
🟢 claude-mama
```

или:

```
🔴 claude (мёртв)
🟢 claude-mama
```

#### `/restart`

Перезапускает основную screen-сессию через `claude-bot.sh restart-main`. Полезно когда сессия мертва и нет возможности сделать это из Telegram основного бота.

#### `/restart_mama`

То же самое для Вовки (`restart-mama`).

### Типы алертов

#### Алерт о зависании

Приходит когда launchd-boot.sh обнаруживает что `prompt-active.json` старше 15 минут:

```
⚠️ Claude не отвечает 22 мин — my-project

Возможно, ждёт разрешения или завис.
```

Кнопки:

| Кнопка | Действие |
|---|---|
| 🛑 Прервать (Esc) | Отправляет Escape в screen — прерывает зависший ввод |
| 🔄 Перезапустить | Отправляет Esc, затем `/restart` в screen |

#### Уведомление о завершении хода

После каждого хода Claude через Stop-хук:

```
✅ Готово — Projects

Тесты прошли (12/12). Коммит создан: "fix: auth token validation".
```

Кнопки ▶ / 🔁 / 🛑.

#### 🔄 Перезапуск сессии

При автоматическом восстановлении (launchd обнаружил мёртвую сессию):

```
🔄 Билд-сервер перезапустился
```

#### 🔴/🟢 Онлайн/Офлайн

При `/newsession`:

```
🔴 Билд-сервер: перезапуск — ухожу в оффлайн на ~15 сек
```

```
🟢 Билд-сервер: снова онлайн
```

### Почему контрол-бот независим

Контрол-бот намеренно запускается как отдельный процесс вне screen-сессии:

1. **Работает без кредитов Claude** — не требует API Anthropic, только Telegram Bot API
2. **Переживает смерть сессии** — если screen-сессия упала, контрол-бот продолжает принимать команды и поднимать сессию
3. **Нет конфликта поллеров** — у основного бота уже есть long polling от плагина, второй поллер на том же токене конфликтовал бы
4. **Inline-кнопки** — привязаны к боту, который их отправил. Контрол-бот отправляет и обрабатывает свои кнопки сам.
